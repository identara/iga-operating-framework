# Lifecycle Definition Checklist

*Companion worksheet for chapter 5 (Process). Work through each block; every unchecked box is a defined gap. Statements referenced: PS1 to PS3, PS5 to PS9.*

**Organization:** [name]  **Version:** [n]  **Date:** [date]  **Owner:** [role]

## Joiner (PS1, PS2)

- [ ] The event that triggers a joiner is defined and comes from the authoritative source
- [ ] Birthright access is defined by written policy
- [ ] Everything outside birthright arrives through a request
- [ ] An accountable owner is named for each step

## Mover (PS1)

- [ ] The event that triggers a mover is defined
- [ ] The mover event re-evaluates existing access rather than only granting new access
- [ ] An accountable owner is named for each step

## Leaver (PS1, PS5)

- [ ] The event that triggers a leaver is defined
- [ ] A time bound from departure to revocation is set: [duration]
- [ ] Revocation is verified in the target system, and the record shows it
- [ ] Voluntary and involuntary departures carry distinct paths and time bounds: [durations]

## Leave of absence and return (PS8)

- [ ] Leave suspends access, and the suspension trigger comes from the authoritative source
- [ ] Return requires verification before access is restored
- [ ] Access active during recorded leave surfaces as a defect

## Rehire (PS8)

- [ ] A rehire is treated as a joiner with history rather than a reactivation
- [ ] Prior-tenure entitlements require new justification before restoration

## Conversion and extension (PS8, C2)

- [ ] Conversion between populations re-baselines access, and the prior population's grants do not carry over by default
- [ ] End-dated identities are extended only with re-justification, and the new end date is recorded

## Post-departure disposition (PS8)

- [ ] Disposition is defined: disable, retention period, deletion
- [ ] Ownership of data and dependants transfers at departure

## Request path and emergency access (PS3, PS9)

- [ ] Intake is defined and is the only path
- [ ] Approval chain depth follows tier (chapter 4, P2)
- [ ] Each stage carries a time bound
- [ ] Reconciliation detects grants made outside the path, and each is reversed or regularized
- [ ] Emergency access is defined in advance: who may invoke it, for which systems, through what mechanism
- [ ] Every emergency use is logged at invocation, expires automatically, and receives a post-use review

## Justifications (PS2)

- [ ] Every grant records who approved, on what basis, tied to what business reason

## Exceptions (PS6)

- [ ] Every exception has an expiry
- [ ] Every exception has an owner
- [ ] Renewal requires re-justification
- [ ] An exception register exists and is current

## Non-human lifecycle (PS7)

- [ ] Creation passes a gate that assigns a human owner (chapter 2, O7)
- [ ] Ownership transfers when the holder departs
- [ ] Credential rotation runs on a defined cycle: [cycle]
- [ ] Decommission retires the credential and verifies its dependants
