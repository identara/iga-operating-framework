# Changelog

## v0.1 (draft) · 2026-07-07

Initial public draft.

- Operating core: six chapters in dependency order (Mandate, Ownership, Scope, Prioritization, Process, Cadence) carrying 47 numbered decisions (M1 to M9, O1 to O8, S1 to S7, P1 to P6, PS1 to PS9, C1 to C8), failure modes with observable signals including the lifecycle set (chapter 5, F7 to F9), starting-state and archetype modulation, per-chapter non-normative examples, and the wider process surface (section 5.7).
- Operational Metrics Specification transcribed from Measuring What Moves (SSRN, DOI 10.2139/ssrn.6842545): four metrics, twelve MS statements, a conformance clause, and a non-normative operational telemetry catalogue. No benchmark thresholds by design.
- Archetype profile specification (AP1 to AP5); no profiles authored yet.
- Worked example: Fernway Software, fictional, all six layers with every worksheet filled (`examples/`).
- Nine companions: the charter template, seven worksheets including the platform capability checklist, and the interactive explorer, with Word versions of all eight fillable artifacts in `companions/docx/`.
- Twelve figures as editable SVG; coverage map in the README.
- Repository governance (GOVERNANCE.md), profile template, issue templates, citation metadata, and the lint gate in tools/.
- Chapter 5 statement prefix renamed from PR to PS, avoiding collision with pull request in repository discussion.

Open at this version: Trust Gradient and Justification Half-Life layer mappings remain proposed (core sections 4.5 and 5.5), pending empirical confirmation against pilot data; candidate statement PS10, a maintained process inventory, under consideration.
