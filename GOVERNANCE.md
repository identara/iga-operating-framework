# Governance

*v0.1 status: single maintainer (Vidyaa Ganesh, Identara). This file defines how the framework's own text changes, and it changes by its own normative process.*

## Change classes

**Editorial**: typos, formatting, link repairs, figure regeneration. The maintainer merges these directly, and the commit history is the record.

**Normative**: any change to a numbered statement, a failure mode, a metric definition, the conformance clause, or the profile specification. A normative change requires an issue describing the change by statement identifier, a recorded disposition, a CHANGELOG entry, and a version bump.

## Versioning

Semantic versioning, as stated in the README: additive changes that do not break an existing conformance claim move the minor version, breaking changes move the major version, and editorial changes move a patch version. The changelog records every version.

## Ratification and the review panel

Nothing in v0.x is ratified. Ratification of a 1.0 requires review by a named panel of external practitioners, recorded through repository issues, and the panel will be listed here as it is recruited during the v0.x review period. Until then, the honest description of this framework is a single-author draft under open review.

## The proposed metric mappings

Trust Gradient on Prioritization and Justification Half-Life on Process are confirmed empirically or not at all: the threshold is pilot data showing the metric moving with the layer's decisions, per the Operational Metrics Specification. The maintainer does not confirm them by declaration.
